What is AcornCSS?

Version: 0.3

Status: Production ready. I am currently using it on a live site.

To Do: A whole lot of organisation and tidying needed. Some updating needed.

Credits: This work is a compilation of others work and some original work over many years. I would like to credit others work, but I dont know who created what...

CSS and JS are inserted into this html page, mainly because i am developing on Android. Also so that the CSS and JS files are not remotely called from here. You can move them to separate files on your webspace if you want.

    Features

  • Designed Mobile First - Mobile First Design means that you won't find any "hover" CSS in this framework. Touch screens don't use hover. Mobile First means designing for portrait mode with fallback for landscape orientation. If in landscape mode the screen width is less than 805px then AcornCSS assumes you are using a simple mobile or watch and keeps the design in Portrait mode.
  • Small - AcornCSS does not change browser default unless necessary. AcornCSS only includes frequently needed functions.
  • Easy to Remember - We use Structural Naming Conventions. All classnames describe what the class is FOR. For instance the classname "clearFloat" clears a float, the classname "width-50" makes an object 50% of its parent (generally the viewport). This way classnames become very easy to remember. We use camelCase where classnames are all letters and a hypen to separate numbers. This benefits ease of reading without lengthening the classname and increasing the size of the CSS file.
  • Themed - We use one colour for theming - and 3 lighter shades and 3 darker shades of that colour. Since our themes are basically one colour - you never have colours clashing. At the end of the day simpler is better. You just have to enter the Hue and Saturation of your favourite color. Check your favourite search engine for "hsl color conversion".

This page demonstrates AcornCSS, as it is designed Mobile First - you should be viewing it in a mobile.

AcornCSS Typography

Default font size is 100% - in other words whatever is the default in the current browser. Most other frameworks use pixels and on mobile it's too small and on desktop too big. 100% is the perfect size to read - turn your mobile to portrait and see the text become smaller, then bigger when you turn back to portrait.

All font sizes are based on rem, not pixels not em. this means they scale properly. Classname "font-12" is 20% bigger (1.2rem) or 120% of default browser font size.

Most other frameworks build on what has gone before and include all sorts of obsolete junk (like using pixels for font size). We have designed AcornCSS from the start for mobile devices.

Support FAQs

How do I transfer a Domain Name?
Login to your current registrar's control panel. Check that the domain does not have Domain Privacy and is not locked. Copy the Domain Password. Go to your new registrar and click the transfer in link and enter the domain name to be transferred and the domain password. That's it.
How do I use DNS?
Firstly you should only use your Domain Name Registrars DNS as the last option. You should use your Website Hosting DNS as first preference and Email Hosting DNS as second preference. If neither of these provide DNS then you are welcome to use ours. Firstly click the link to manage the Domain Name you want to use DNS. Click the dropdown to park the Domain Name. This gives you the correct Nameservers. Now edit your DNS, you can add A records, MX records and other record types too. You get the correct records from your hosting and email providers. DNS services are self serve.
